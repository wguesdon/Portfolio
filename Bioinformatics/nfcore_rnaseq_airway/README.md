# nf-core RNA-seq & Differential Expression Pipeline

End-to-end RNA-seq workflow using [nf-core/rnaseq](https://nf-co.re/rnaseq) and
[nf-core/differentialabundance](https://nf-co.re/differentialabundance), running on
**AWS Batch Spot instances** via Wave + Fusion.

**Dataset:** [GSE52778](https://www.ncbi.nlm.nih.gov/geo/query/acc.cgi?acc=GSE52778) — airway
smooth muscle cells treated with dexamethasone (Himes et al. 2014)

---

## Biological Context

Glucocorticoids such as dexamethasone are front-line treatments for asthma. Himes et al. used
RNA-seq to characterise the transcriptional response of primary human airway smooth muscle cells
from four donors to dexamethasone treatment. The dataset is widely used as a reference example
in Bioconductor DESeq2 tutorials.

| Attribute | Value |
|-----------|-------|
| Organism | *Homo sapiens* |
| Tissue | Airway smooth muscle cells |
| Cell lines (donors) | N61311, N052611, N080611, N061011 |
| Conditions | untreated vs dexamethasone |
| Replicates | 4 per condition (paired cell-line design) |
| Read length | 75 bp paired-end |
| Subsampled to | 1M read pairs per sample |

## Pipeline Overview

```
FASTQ files (SRA: GSE52778)
        │
        ▼
┌─────────────────────────────────┐
│  nf-core/rnaseq v3.14.0         │
│  • FastQC + MultiQC             │
│  • Trim Galore                  │
│  • Salmon pseudo-alignment      │
│  • DESeq2 QC (PCA, distances)   │
└──────────────┬──────────────────┘
               │  salmon.merged.gene_counts.tsv
               ▼
┌─────────────────────────────────┐
│  nf-core/differentialabundance  │
│  v1.4.0                         │
│  • DESeq2 (dex vs untreated)    │
│  • Volcano / MA plots           │
│  • g:Profiler pathway analysis  │
│  • Interactive Shiny report     │
└─────────────────────────────────┘
```

**Execution:** AWS Batch Spot instances (c5/m5 families, us-east-1)
**File I/O:** Seqera Wave + Fusion — S3 mounted as a local filesystem inside containers (no AWS CLI required in images)

## Quick Start

### Prerequisites

- [Nextflow](https://nextflow.io/) ≥ 24.x
- Docker (for local runs) or AWS credentials configured (for AWS Batch)
- AWS CLI (`aws sts get-caller-identity` succeeds)
- sra-tools (`conda create -n sra-tools -c bioconda sra-tools`)

### 1. Install Nextflow

```bash
make install-nextflow
```

### 2. Download the airway dataset

```bash
conda activate sra-tools
make data-airway
```

Downloads ~3–4 GB: 8 paired-end FASTQ files + GENCODE v44 references.

### 3a. Run locally (Docker)

```bash
# RNA-seq quantification
make test-rnaseq-airway

# Differential expression (requires rnaseq results)
make test-da-airway

# Or both in sequence
make workflow-airway
```

### 3b. Run on AWS Batch (Spot instances)

```bash
# Upload data to S3
export S3_BUCKET=your-bucket-name
make upload-airway

# RNA-seq on Batch
ENV=aws S3_BUCKET=$S3_BUCKET make test-rnaseq-airway

# Differential expression on Batch (uses count matrix from S3 results)
ENV=aws S3_BUCKET=$S3_BUCKET make test-da-airway
```

Results land at `s3://${S3_BUCKET}/results/`.

See [docs/aws-batch-setup.md](docs/aws-batch-setup.md) for the full infrastructure setup guide.

### Resume a run

```bash
# Local
./scripts/run.sh --pipeline rnaseq --dataset airway --env local --resume

# AWS
S3_BUCKET=$S3_BUCKET ./scripts/run.sh --pipeline rnaseq --dataset airway --env aws --resume
```

## AWS Architecture

```
Local machine (Nextflow head node)
        │
        │  submits jobs
        ▼
  AWS Batch Job Queue (nextflow-queue)
        │
        │  provisions
        ▼
  Compute Environment (nextflow-ce)
  • Type: MANAGED SPOT
  • Instance families: c5, m5
  • vCPU range: 0–256
  • Allocation: SPOT_CAPACITY_OPTIMIZED
  • Custom AMI: ECS-optimized AL2 (100 GB gp3 root)
        │
        │  reads/writes via Fusion FUSE
        ▼
  S3 Bucket (nf-core-tests-wg)
  ├── data/airway/        ← inputs
  ├── work/               ← Nextflow work dir (intermediate files)
  └── results/            ← final outputs
```

Wave + Fusion eliminates the need for an AWS CLI inside Docker containers:
Wave augments images at runtime; Fusion mounts S3 as a POSIX filesystem via FUSE.
